# Multi-Agent Indian Courtroom RL Environment ⚖️

> **"Build a Mini RL Environment with defined tasks, graders, and reward logic."**

## 💡 Problem Statement
Legal reasoning is one of the most complex domains for AI. This project introduces a **Multi-Agent Courtroom Simulation** where an AI agent acts as a lawyer in an Indian courtroom setting. 

The environment aligns with **OpenEnv** standards and tests the agent's ability to:
1. Navigate varying degrees of case complexity (Easy, Medium, Hard).
2. Synthesize arguments based on restricted **State Facts**.
3. Correctly reference and deploy specific **Evidence** to maximize their legal standing.
4. Adhere to formatting guidelines (JSON output) to successfully interact with the Environment API.

## 🧩 Environment Features

### State Space
The environment's observation (returned via `/state` and `/step`) consists of:
- `case_facts` (str): Background information about the dispute.
- `evidence` (List[str]): A list of all available evidence documents.
- `history` (List[str]): Full transcript of the ongoing trial.
- `judge_response` (str): Feedback from the environment (acting as the Judge).
- `current_turn` (int): Number of steps passed.

### Action Space
The environment expects the agent to submit actions as structured JSON containing:
- `argument` (str): The logical synthesis of the lawyer's point.
- `evidence_referenced` (str | null): The exact string match of the evidence being presented to the court.

### 🏆 Meaningful Reward Logic
Rewards are granted on a scale from **0.0 to 1.0**. We use a hybrid grading system:
1. **Deterministic Partial Rewards (per step)**:
   - **+0.1**: For presenting valid evidence from the list.
   - **+0.3**: For presenting the *crucial* piece of evidence required to win the specific task.
   - **-0.1**: For hallucinating evidence not present in the record.
   - **+0.05**: For formulating a substantive argument (>10 words).
2. **LLM Evaluation (Episode End)**:
   - Up to **+0.5** bonus points generated by a Judge LLM model that reviews the transcript for logical consistency and persuasiveness (only awarded if basic deterministic checks pass).

## 🚀 Tasks
We have implemented 3 escalating tasks (via `tasks.py`):
1. **[Easy] case-001-easy**: Contract dispute requiring a basic evidence match.
2. **[Medium] case-002-medium**: Theft accusation requiring filtering irrelevant witnesses and finding an alibi.
3. **[Hard] case-003-hard**: Corporate fraud requiring specific multi-step reasoning to tie an email chain to a ledger.

## 🛠️ How to Run & Setup

### 1. Run the FastAPI Server Locally
The environment runs as a standard Hugging Face Space FastAPI container.
```bash
pip install -r requirements.txt
python env.py
```
*(Runs on port 7860 by default)*

### 2. Run the Baseline Inference
Our `inference.py` script rigorously follows the Hackathon's `[START]`, `[STEP]`, and `[END]` output formatting.
```bash
export HF_TOKEN="your_token" # (Optional: Used for LLM grading and Agent decision making)
python inference.py
```

### 3. Submission Validation
Check compliance using the provided validation script:
```bash
bash scripts/validate-submission.sh <YOUR_SPACE_URL> .
```
